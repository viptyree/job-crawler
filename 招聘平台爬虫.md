# 招聘平台爬虫

可以。下面是基于：

```text
Codex job\kaifa\Goodjob\job-crawler
```


# 第 4 步：新增公司别名生成能力

目标：

用户输入公司全称后，自动生成几个搜索候选词。

例如：

```text
广州云启未来科技有限公司
```

生成：

```text
广州云启未来科技有限公司
云启未来科技
云启未来
```

要做：

```text
1. 实现 alias_generator.py
2. 支持去掉有限公司、有限责任公司、股份有限公司
3. 支持去掉科技、信息技术、网络科技等弱后缀
4. 支持去掉广州、深圳、上海、北京等地区前缀
5. 提供后端测试接口
```

新增接口：

```text
POST /api/v1/company-intel/aliases/generate
```

请求：

```json
{
  "company_name": "广州云启未来科技有限公司"
}
```

返回：

```json
{
  "aliases": [
    "广州云启未来科技有限公司",
    "云启未来科技",
    "云启未来"
  ]
}
```

验收标准：

```text
1. 输入公司名能返回别名
2. 空输入能给出错误提示
3. 重复别名会自动去重
4. 不影响原项目任何功能
```

```json
  "source_url": "https://example.com/job/1",
  "match_type": "mock"
}
```

验收标准：

```text

2. 能通过后端调用 zhilian 模拟查询
```

# 第 6 步：新增公司查询主接口

目标：

完成“输入公司名，选择平台，生成查询任务，保存查询结果”的后端主流程。

新增接口：

```text
POST /api/v1/company-intel/search
GET /api/v1/company-intel/queries/{query_id}
GET /api/v1/company-intel/queries/{query_id}/jobs
```

流程：

```text
1. 接收公司名称、平台、城市、岗位关键词
2. 创建或复用公司记录
3. 自动生成公司别名
4. 创建 company_intel_queries 查询任务
5. 调用多个平台适配器
6. 保存 company_intel_jobs
7. 更新查询任务状态
8. 返回 query_id
```

验收标准：

```text
1. 调用 search 接口能创建查询任务
2. 查询任务能保存到 company_intel_queries
3. 模拟岗位能保存到 company_intel_jobs
4. 能通过 query_id 查询任务状态
5. 能通过 query_id 查询岗位列表
```

# 第 7 步：新增招聘活跃度评分

目标：

查询完成后，自动生成招聘活跃度评分。

评分规则第一版：

```text
在招岗位数量：30分
平台覆盖数量：20分
城市覆盖数量：10分
最近岗位数量：20分
岗位类型丰富度：10分
薪资完整度：10分
```

新增文件：

```text
backend/app/company_intel/scorer.py
```

新增接口：

```text
GET /api/v1/company-intel/companies/{company_id}/score
GET /api/v1/company-intel/queries/{query_id}/score
```

评分结果：

```json
{
  "score": 82,
  "level": "高活跃",
  "reason_text": "该公司在多个平台存在招聘信息，岗位数量较多，近期可能处于扩张阶段。"
}
```

验收标准：

```text
1. 查询完成后能生成评分
2. 评分保存到 company_intel_scores
3. 能按 query_id 查询评分
4. 能按 company_id 查询最新评分
```
