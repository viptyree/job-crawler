# 企业招聘情报第二阶段测试指南

本指南用于测试 `company_intel` 第二阶段能力：平台账号登录窗口、登录状态检测、真实优先查询、仅真实查询、模拟降级和报告导出。

## 1. 启动后端

在项目根目录打开 PowerShell：

```powershell
cd backend
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

看到类似输出后保持窗口打开：

```text
Uvicorn running on http://0.0.0.0:8000
```

浏览器访问：

```text
http://localhost:8000/health
```

预期返回：

```json
{"status":"ok","service":"job-crawler"}
```

## 2. 启动前端

新开一个 PowerShell：

```powershell
cd frontend
pnpm dev
```

浏览器打开：

```text
http://localhost:5173
```

如果 5173 被占用，以终端实际显示的地址为准。

## 3. 先测模拟数据链路

打开页面：

```text
企业招聘情报 -> 公司查询
```

填写：

```text
公司名称：广州云启未来科技有限公司
查询平台：BOSS直聘、智联招聘、前程无忧、拉勾网
城市：广州
岗位关键词：销售经理
查询模式：模拟数据
```

点击：

```text
开始查询
```

预期结果：

- 页面提示查询完成。
- 下方出现岗位结果表格。
- 出现招聘活跃度评分。
- 每个平台应有模拟岗位数据。

再打开：

```text
企业招聘情报 -> 查询任务
```

预期结果：

- 能看到刚才创建的查询任务。
- 状态为 `success`。
- 岗位数大于 0。

## 4. 测试公司档案

打开：

```text
企业招聘情报 -> 公司档案
```

预期结果：

- 能看到刚才查询过的公司。
- 能看到最新评分。

点击公司右侧：

```text
查看
```

预期结果：

- 进入公司详情页。
- 能看到历史查询任务。
- 点击历史查询的「报告」可进入报告页。

## 5. 测试报告和 Excel 导出

打开：

```text
企业招聘情报 -> 查询任务
```

点击某个任务：

```text
查看结果
```

预期结果：

- 进入招聘报告页。
- 能看到任务信息、活跃度评分、岗位明细。

点击：

```text
导出 Excel
```

预期结果：

- 浏览器下载 `company_intel_query_{query_id}.xlsx`。
- Excel 中包含公司名称、平台、岗位名称、城市、薪资、经验、学历、原始链接、匹配方式、查询时间。

## 6. 测试平台账号登录窗口

打开：

```text
企业招聘情报 -> 平台账号
```

对某个平台点击：

```text
打开登录
```

预期结果：

- 本机会弹出一个 Chromium 浏览器窗口。
- 浏览器进入对应招聘平台登录页。
- 在浏览器中手动完成登录。
- 如果出现验证码、安全验证、扫码登录，按平台页面提示手动处理。

处理完成后不要清理浏览器数据。这个浏览器资料会保存在：

```text
backend/data/company_intel_sessions/{platform}
```

## 7. 测试登录状态检测

在「平台账号」页，对刚登录的平台点击：

```text
检测
```

可能结果：

```text
可能已登录
```

说明当前本地浏览器会话大概率可用于真实查询。

也可能结果：

```text
未登录
检测失败
缺浏览器
```

含义：

- `未登录`：平台仍要求登录或验证码，需要重新点「打开登录」处理。
- `检测失败`：平台网络、风控或页面加载失败。
- `缺浏览器`：Playwright 浏览器未安装。

如果提示缺浏览器，在 `backend` 目录执行：

```powershell
python -m playwright install chromium
```

然后重启后端再测。

## 8. 测试真实优先模式

打开：

```text
企业招聘情报 -> 公司查询
```

填写：

```text
公司名称：广州云启未来科技有限公司
查询平台：选择刚才登录过的平台
城市：广州
岗位关键词：销售经理
查询模式：真实优先
```

点击：

```text
开始查询
```

预期结果分两种：

第一种：真实查询成功

- 岗位结果出现。
- 岗位的匹配方式可能显示 `real`。

第二种：真实查询失败但自动降级

- 页面仍显示岗位结果。
- 岗位匹配方式显示 `mock`。
- 查询任务详情里的 `error_message` 会记录类似：

```text
boss: 登录状态检测失败，可能需要手动打开平台确认。
```

这是正常结果，表示真实平台暂时不可用，但演示链路没有断。

```text
企业招聘情报 -> 公司查询
```

填写：

```text
公司名称：广州云启未来科技有限公司
查询平台：选择一个平台
城市：广州
岗位关键词：销售经理
查询模式：仅真实
```

点击：

```text

```

预期结果：

- 如果平台登录和网络都正常，返回真实岗位。
- 如果未登录、验证码、网络阻断或平台风控，任务可能成功但岗位数为 0，并在任务信息中保留提示。

这个模式用于确认真实抓取是否真的可用，不保证演示结果稳定。

## 10. 推荐测试顺序

按下面顺序测最稳：

```text
1. 后端 health
2. 前端首页
3. 模拟数据查询
4. 查询任务
5. 公司档案
6. 报告导出
7. 平台账号打开登录
8. 平台账号检测
9. 真实优先查询
10. 仅真实查询
```

## 11. 常见问题

### 点击打开登录没有弹窗

检查后端终端是否有 Playwright 或 Chromium 相关错误。

可在 `backend` 目录执行：

```powershell
python -m playwright install chromium
```

然后重启后端。

### 检测一直失败

可能原因：

- 当前网络无法访问招聘平台。
- 平台关闭了连接。
- 平台触发安全验证。
- 登录窗口还没有完成登录。

处理方式：

```text
1. 再点一次打开登录
2. 在弹出的浏览器里手动访问平台首页
3. 完成登录或验证码
4. 回到系统点击检测
```

### 真实优先返回 mock

这是预期降级行为，说明真实查询遇到问题，但系统保留了可演示结果。

要确认原因，查看：

```text
企业招聘情报 -> 查询任务 -> 查看结果
```

或调用：

```text
GET /api/v1/company-intel/queries/{query_id}
```

查看 `error_message`。

### 仅真实没有岗位

说明真实平台没有成功返回可解析岗位。常见原因是未登录、验证码、平台风控、页面结构变化或网络失败。

先回到「平台账号」页重新打开登录并检测。

## 12. 测试通过标准

第二阶段基础能力通过标准：

- 模拟数据查询完整可用。
- 查询任务、公司档案、报告页可用。
- Excel 能导出。
- 平台账号页能打开登录窗口。
- 平台账号页能执行登录检测。
- 真实优先模式在真实失败时能降级并保存提示。
- 仅真实模式能暴露真实平台问题，不静默伪装成功。
